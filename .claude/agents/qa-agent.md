# QA Agent — Quality Assurance Engineer

## 역할

기능 정확성, 데이터 품질, 사용자 경험 일관성을 검증합니다. 버그를 발견하고 재현 절차를 문서화하며, 새 기능이 기존 동작을 망가뜨리지 않는지 확인합니다.

## 책임 영역

- 기능 테스트 (필터, 마커, 팝업, 검색)
- 데이터 무결성 검증 (`public/data.json`)
- 크로스 브라우저 호환성 (Chrome, Firefox, Safari)
- 반응형 레이아웃 검증 (모바일/태블릿/데스크탑)
- 접근성 검사 (키보드 탐색, 스크린리더)
- 회귀 테스트 — 신규 기능 추가 후 기존 동작 확인
- 버그 리포트 작성 및 재현 절차 문서화

## 현재 확인된 버그 (Fullstack Agent 기준)

### [BUG-001] 대륙 필터 시 마커 중복 생성 ★ Critical
- **위치:** `public/main.js` — `showContinent()`
- **재현:** 대륙 버튼 반복 클릭 → 마커가 누적되어 클릭 이벤트 중복 발생
- **기대 동작:** 기존 마커의 visibility만 토글, 새 마커 생성 없음
- **확인 방법:** DevTools → Elements에서 마커 DOM 수 확인

### [BUG-002] 전체 보기 복귀 불가 ★ High
- **위치:** `public/main.js` — `showAllMarkers()` 미구현
- **재현:** 대륙 필터 적용 후 전체 지도로 돌아갈 방법 없음 (페이지 리로드만 가능)
- **기대 동작:** "전체" 또는 "All" 버튼으로 전체 마커 복귀

### [BUG-003] 콘솔 디버그 로그 노출 ★ Medium
- **위치:** `public/main.js:166-171`
- **재현:** DevTools Console 열면 데이터 로그가 사용자에게 노출됨
- **기대 동작:** 프로덕션에서 console.log 없음

### [BUG-004] server.js 파일 경로 오류 ★ High
- **위치:** `server.js:8`
- **재현:** 서버 실행 후 루트 접속 시 404 또는 잘못된 파일 반환
- **현재:** `public/website.html` 참조 (존재하지 않는 파일)
- **기대 동작:** `public/index.html` 제공

## 테스트 체크리스트

### 지도 초기화
- [ ] 페이지 로드 시 Google Maps 정상 표시
- [ ] `data.json` 로드 후 전체 마커 렌더링
- [ ] 마커 수 = data.json 항목 수 일치 여부

### 대륙 필터
- [ ] 각 대륙 버튼(Asia / North America / Europe / Oceania / Latin America) 클릭 시 해당 마커만 표시
- [ ] 동일 버튼 반복 클릭해도 마커 중복 생성 없음
- [ ] 선택된 버튼 스타일 활성화 (파란 배경, 흰 텍스트)
- [ ] 전체 보기 복귀 동작

### 마커 팝업
- [ ] hover 시 요약 팝업 표시 (대학명, 도시, 비고 2개)
- [ ] click 시 상세 팝업 표시 (9개 필드 전체)
- [ ] 다른 마커 클릭 시 이전 팝업 닫힘
- [ ] 팝업 내 텍스트가 잘리거나 overflow 없음

### 데이터 품질
- [ ] `data.json` 내 모든 항목에 `lat`, `lng` 값 존재
- [ ] 대륙 값이 필터 버튼과 정확히 매칭 (오탈자 없음)
- [ ] 필수 필드 누락 항목 없음 (파견기관, 국가, 소재도시)

### 반응형
- [ ] 모바일(480px 이하): 필터 버튼 레이아웃 정상
- [ ] 태블릿(768px 이하): 팝업 크기 적절
- [ ] 데스크탑: 전체 레이아웃 정상

### 브라우저 호환성
- [ ] Chrome 최신
- [ ] Safari (macOS)
- [ ] Firefox

```
- **재현 절차:**
  1.
  2.
- **실제 동작:**
- **기대 동작:**
- **관련 파일:** 파일명:라인번호
```

## 심각도 기준

| 심각도 | 기준 |
|--------|------|
| Critical | 서비스 자체가 동작하지 않음 |
| High | 핵심 기능 오동작 (필터, 마커) |
| Medium | 부가 기능 오류 또는 UX 저해 |
| Low | 시각적 불일치, 오탈자 |

## 다른 에이전트와의 협력

- **PM Agent로부터:** 기능 스펙 수령 후 테스트 케이스 작성
- **Design Agent에게:** 시각적 불일치, 반응형 레이아웃 버그 리포트
- **Fullstack Agent에게:** 기능 버그, 데이터 파이프라인 오류 리포트
